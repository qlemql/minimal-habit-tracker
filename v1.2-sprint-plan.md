**작업 순서**:

**검증 기준**:

### Phase C — 영문 번역 (1.5~2일)

### Phase D — 위젯 + 알림 i18n (1.5일)

**중요**: RN i18next는 위젯/알림 외부에서 동작하지 않음. 네이티브 리소스로 별도 처리.

1. **D1 — iOS WidgetExtension**:
   - `targets/HabitWidget/`에 `ko.lproj/Localizable.strings` + `en.lproj/Localizable.strings`
   - Swift 코드의 한국어 하드코딩 제거 → `NSLocalizedString("...", comment: "")`
   - `Info.plist`에 `CFBundleLocalizations: ["ko", "en"]` 추가
   - 검증: 시뮬레이터 언어 ko/en 토글하며 위젯 표시 확인
2. **D2 — Android Widget**:
   - `targets/HabitWidgetAndroid/res/values/strings.xml` (ko 디폴트)
   - `targets/HabitWidgetAndroid/res/values-en/strings.xml`
   - Kotlin 코드의 한국어 하드코딩 제거 → `context.getString(R.string.xxx)`
   - 검증: 에뮬레이터 언어 ko/en 토글하며 위젯 표시 확인
3. **D3 — 알림 카테고리 액션**:
   - `setupNotificationCategories()`의 "완료" 텍스트 i18n
   - 카테고리는 OS 언어 기준 캐싱 → 출시 전 재등록 검증 필요
   - `expo-notifications` 카테고리는 i18next 키 사용 가능 (RN 영역)

**검증 기준**:
- iOS/Android 양쪽 시뮬레이터 언어 변경 후 위젯/알림 텍스트 분기 정상

### Phase E — 메타데이터 + 자산 (1.5일)
