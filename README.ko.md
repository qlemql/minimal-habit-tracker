> *많으면 지치고, 적으면 지켜져요.*

## 왜 만들었나요?

대부분의 습관 앱은 같은 방식으로 실패합니다. 무제한으로 습관을 추가하게 하고, 완벽한 연속 기록을 요구하고, 하루만 빠뜨려도 0으로 리셋합니다. 결국 앱 자체를 안 열게 되고 — 습관을 만들어주려던 앱이 가장 먼저 포기되는 앱이 됩니다.

싹은 반대로 갑니다. **딱 3개 습관. 하루 빠지면 쉼표일 뿐, 마침표가 아닙니다. 모든 입력은 5초 안에 끝.**

## 모든 것을 결정하는 두 가지 설계 원칙

### 1. 3개 제한은 제약이 아니라 기능입니다

행동 연구는 일관됩니다. 한 번에 여러 습관을 만들려는 사람은 결국 아무것도 만들지 못합니다. 하드 제한은 질문을 강제합니다 — *정말 중요한 3개가 무엇인가?* 네 번째를 추가하려면 기존 습관을 하나 빼야 합니다.

### 2. 이어가는 흐름 (Forgiving Streaks)

일반적인 스트릭 로직은 하루 빠지면 카운터가 0으로 리셋됩니다. 그 심리적 효과는 잘 알려져 있습니다 — 하루 빠진 것이 완전한 포기로 이어집니다.

싹에서는 하루 빠진 것이 **흐름 안의 쉼표**입니다. 이틀 연속 빠지면 새로운 흐름이 시작됩니다. 이는 습관 형성에 대한 행동 연구가 실제로 말하는 것과 일치합니다 — 하루 정도의 공백은 장기적 결과에 거의 영향이 없지만, 리셋의 처벌은 습관 자체를 죽일 만큼 가혹합니다.

```
● ● ◌ ● ● ●
● = 완료    ◌ = 쉼표 (흐름 유지)
```

> 쉼표가 있어도 문장은 이어지니까요.

## 주요 기능

**핵심**

- 3개 습관 제한 — 의도적 설계
- 원탭 체크인 (열고 → 탭하고 → 닫기)
- 이어가는 흐름 시스템
- 카드 길게 누르면 이름·아이콘·알림 수정

**네이티브 위젯**

- iOS 잠금 화면 위젯 3종: 시계 위 한 줄, 진행률 링, 도트 + 카운트
- iOS 홈 화면 위젯
- Android 홈 화면 위젯

**디테일**

- 습관별 알림과 스마트 기본 시간 제안
- 싹 성장 단계 애니메이션: 씨앗 → 발아 → 떡잎 → 줄기 → 꽃봉오리 → 개화
- 라이트·다크·시스템 모드 (다크가 기본)
- 해금 시스템: 흐름이 자라면 새 아이콘과 색상이 열림

**프라이버시**

- 모든 데이터는 기기에만 저장
- 서버 없음. 계정 없음. 추적 없음.

## 기술적으로 흥미로운 부분

**Expo 앱에서의 iOS 위젯**은 간단하지 않습니다 — Expo는 위젯 익스텐션을 기본 지원하지 않거든요. [`@bacons/apple-targets`](https://github.com/EvanBacon/expo-apple-targets)와 커스텀 config 플러그인(`plugins/withSharedDefaults.js`)을 만들어 Swift 타겟을 iOS 빌드에 주입하는 방식으로 해결했습니다.

**App Group을 통한 네이티브 ↔ JS 데이터 동기화**로, IPC 없이 위젯이 상태 변화를 즉시 반영합니다.

**`react-native-reanimated` 4 worklets**로 스트릭 전환 애니메이션이 UI 스레드에서 실행되어, 구형 기기에서도 60fps를 유지합니다.

## 프로젝트 구조

```
app/                  Expo Router 화면 (index, add, edit, settings, onboarding)
src/
├── components/       UI 프리미티브 및 습관 카드
├── store/            Zustand 스토어
├── utils/            스트릭 계산, 날짜 헬퍼
├── types/            공유 TypeScript 타입
└── constants/        테마 토큰, 기본 습관
modules/
└── shared-defaults/  App Group 동기화 네이티브 모듈
plugins/
└── withSharedDefaults.js  커스텀 Expo config 플러그인
targets/
└── HabitWidget/      Swift 위젯 익스텐션
```
