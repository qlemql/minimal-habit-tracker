## 0. 결정사항 스냅샷 (2026-05-13)

### 핵심 가치
- "습관이 자리잡았다"는 자기 인식을 시스템에 반영
- 단순 삭제와 졸업 차별화 (삭제 = 흔적 없음 / 졸업 = 정원에 보존)
- 활성 3개 슬롯 + 졸업한 정원(별도 컬렉션) 구조 — 룰 1 신성불가침 유지

### 임계값 (시스템 고정)
- **0~6일** (씨앗/발아): 졸업 불가, `[삭제]` 만
- **7일** (떡잎부터): 조기 졸업 가능, `[조기 졸업]` 노출
- **50일** (개화): 정시 졸업, `[졸업시키기]` 노출 + 자동 제안 다이얼로그 1회

### Pro 재정렬
- 원래 v1.3 = Pro IAP + 통계 화면 → **v1.3 = 졸업 + 통계(Free)** 로 재정의
- Pro IAP → v1.4로 이연
- 유료 팩 → v1.5로 이연

## 1. 데이터 모델

### Habit 인터페이스 확장

```typescript
interface Habit {
  // 기존 필드 유지
  id: string;
  name: string;
  icon: string;
  color: string;
  createdAt: string;
  reminderTime?: string;

  // 신규 (graduation)
  graduatedAt?: string;        // YYYY-MM-DD
  graduatedStage?: GrowthStageId;  // 'leaf' | 'stem' | 'bud' | 'bloom'
  totalFlowDays?: number;      // 졸업 시점 누적 흐름 일수
  isGraduated?: boolean;       // 활성 슬롯 카운트 제외 플래그
}
```

### 마이그레이션
- 기존 사용자 영향 0 — 신규 필드는 모두 optional
- `isGraduated` 미정의 = `false` 취급
- 기존 Habit 데이터 그대로 사용 가능

### 활성 슬롯 카운트 변경
- `habits.filter(h => !h.isGraduated)` 로 활성 카운트
- 룰 1 (3개 제약) = 활성 슬롯 3개 — 졸업한 항목은 카운트 제외
- 새 습관 추가 시 활성 카운트 ≥ 3 이면 차단 (기존 동작 동일)

### 자동 제안 트래킹
- `useSettingsStore` 또는 별도 store에 `graduationProposedFor: Record<HabitId, boolean>` 추가
- 한 습관당 자동 제안 1회 — 거절 후 다시 안 묻기

## 2. UI/UX 변경

### 편집 화면 (`app/edit.tsx`)

**동적 액션 노출**:
```typescript
const showGraduate = habit.totalFlowDays >= 7 || currentFlow >= 7;
const isFullGraduation = currentFlow >= 50;
const graduateLabel = isFullGraduation
  ? t('graduation.action.graduate')      // "졸업시키기" / "Graduate"
  : t('graduation.action.earlyGraduate'); // "조기 졸업" / "Graduate early"
```

### 자동 제안 다이얼로그 (`app/index.tsx` 또는 별도 컴포넌트)
- 50일 도달 시점 다음 체크 직후 발동 (체크 액션 마무리 → 다이얼로그)
- 한 습관당 1회 — `graduationProposedFor[habit.id] = true` 마킹
- 거절(`[계속 키울게요]`) 후 다시 안 묻기

### 조기 졸업 확인 다이얼로그
- 50일 미만 졸업 시 한 번 더 확인
- "조기 졸업할까요? 개화까지 N일 남았어요"

### 통계 화면 (`app/stats.tsx` 신규)
- 진입: `app/settings.tsx`에서 새 항목 "통계 보기" → router.push('/stats')
- 구조:
  ```
  [헤더: 통계]
  [기본 통계 카드: 활성 습관 / 총 완료 / 최장 흐름 / 해금 수]
  [졸업한 정원 섹션 제목]
  [졸업 항목 카드들 (이름/아이콘/졸업 일자/총 흐름/단계 chip)]
  [빈 상태: "아직 졸업한 습관이 없어요"]
  ```
- 기존 settings의 stats card는 유지 (요약 형태) — 통계 화면에서 더 자세히

### Settings 진입 링크 추가
- 기존 stats card 아래 또는 옆에 "통계 보기" 진입 버튼
- 룰 3 준수: 메인 화면 변경 0개, settings에만 추가
