## 1. 프로젝트 개요

기존 습관 트래커 앱들(Habitify, Habitica, Productive 등)은 기능이 너무 많고 복잡해서 오히려 사용자가 이탈하는 문제가 있다. 이 앱은 "딱 3개 습관만 집중"이라는 제약을 컨셉으로, 극도로 단순한 습관 관리 경험을 제공한다.

핵심 가치는 **미니멀리즘**이다. 적게 추적할수록 더 잘 지키게 된다는 심리학적 원리를 활용한다.

## 2. 핵심 콘셉트

### 문제 정의

- 기존 습관 앱은 습관을 무제한으로 추가할 수 있어 오히려 부담이 됨
- 설정할 것이 많고 (알림, 카테고리, 통계) 진입 장벽이 높음
- 결국 앱 자체를 안 열게 되는 악순환

### 솔루션

- 무료 기준 최대 3개 습관만 등록 가능 (의도적 제약)
- 앱을 열면 오늘 할 일 3개가 바로 보임
- 원탭으로 완료 체크 → 끝
- 홈 위젯으로 앱을 열지 않아도 체크 가능

### 왜 3개인가?

- 행동 심리학에서 습관 형성의 핵심은 "작게 시작하기"
- 3개는 인지 부하 없이 기억할 수 있는 최적의 수
- 제약이 곧 차별점 — "뭘 뺄까"를 고민하게 만드는 설계

## 3. 핵심 기능

### Free 기능

| 기능 | 설명 |
|------|------|
| 습관 등록 | 최대 3개, 이름 + 아이콘 + 색상 선택 |
| 원탭 체크 | 오늘의 습관을 탭 한번으로 완료 표시 |
| 스트릭 표시 | 연속 달성 일수 카운터 |
| 주간 캘린더 | 이번 주 달성 현황 한눈에 보기 |
| 홈 위젯 | 앱을 열지 않고 위젯에서 바로 체크 |
| 알림 | 습관별 리마인더 알림 설정 |

### Pro 기능 (1회 결제)

| 기능 | 설명 |
|------|------|
| 무제한 습관 | 3개 제한 해제 |
| 월간/연간 통계 | 달성률 그래프, 최장 스트릭 기록 |
| 테마 커스터마이징 | 다크/라이트 + 커스텀 색상 |
| 데이터 백업 | 클라우드 동기화 (Supabase) |
| 습관 아카이브 | 완료된 습관 보관 및 재활성화 |

## 4. 사용자 플로우

### 온보딩 (최초 실행)

```
① 앱 실행
② "딱 3개만 골라보세요" 안내 화면
③ 습관 3개 등록 (프리셋 선택 or 직접 입력)
④ 알림 시간 설정 (선택)
⑤ 메인 화면 진입
```

### 일상 사용

```
① 앱 실행 (또는 위젯)
② 오늘의 3개 습관 확인
③ 완료한 습관 탭하여 체크
④ 3개 모두 완료 시 축하 애니메이션
⑤ 끝 (평균 소요 시간: 5초)
```

### 앱 핵심 화면

| 화면 | 설명 | 핵심 요소 | 우선순위 |
|------|------|-----------|---------|
| 메인 (오늘) | 오늘의 습관 3개 + 체크 | 원탭 체크, 스트릭 카운터 | P0 |
| 주간 뷰 | 이번 주 달성 현황 | 7일 캘린더 그리드 | P0 |
| 습관 편집 | 습관 추가/수정/삭제 | 아이콘, 색상, 알림 설정 | P0 |
| 통계 (Pro) | 월간/연간 달성률 | 그래프, 최장 스트릭 | P1 |
| 설정 | 테마, 백업, 구매 | 1회 결제 화면 | P1 |
| 홈 위젯 | 앱 밖에서 체크 | 3개 습관 + 체크 버튼 | P0 |

## 5. 디자인 방향

### 디자인 원칙

- **극도의 단순함**: 메인 화면에 습관 3개와 체크 버튼만 표시
- **여백 중심**: 정보 밀도를 최소화하고 여백을 충분히 확보
- **부드러운 인터랙션**: 체크 시 만족감을 주는 햅틱 피드백 + 애니메이션
- **다크 모드 우선**: 아침/밤에 주로 사용하므로 다크 모드 기본

### 비주얼 레퍼런스

- Things 3 (미니멀 태스크 관리)
- Streaks (원형 진행률)
- iOS 기본 시계 앱 (깔끔한 인터페이스)

### 기술 스택

| 항목 | 기술 |
|------|------|
| 모바일 앱 | React Native / Expo |
| 로컬 저장소 | AsyncStorage (기본) |

```

  habitId: string
  date: string (YYYY-MM-DD)
```

### 아키텍처 특징

- **로컬 퍼스트**: 기본적으로 모든 데이터는 디바이스에 저장 (오프라인 완전 지원)
- **선택적 동기화**: Pro 유저만 Supabase 클라우드 백업 활성화
- **서버리스**: 별도 백엔드 서버 없음 → 운영 비용 최소화

## 7. 수익화 전략

### 가격 모델: 1회 결제 (Lifetime)

| 티어 | 가격 | 포함 기능 |
|------|------|----------|
| Free | ₩0 | 습관 3개, 원탭 체크, 스트릭, 주간 뷰, 위젯, 알림 |
| Pro | ₩6,500 ($4.99) 1회 결제 | 무제한 습관, 월간/연간 통계, 테마, 클라우드 백업, 아카이브 |

### 왜 1회 결제인가?

- 습관 앱은 "매일 열어야 하는" 앱 → 구독 피로감이 이탈로 이어짐
- 1회 결제는 구매 허들이 낮고, "한번 사면 끝"이라는 심리적 안정감 제공
- 리뷰/평점에 긍정적 영향 → 앱스토어 랭킹에 유리
- 이후 추가 기능팩 (위젯 테마, 아이콘 팩 등)으로 추가 수익 가능

| Productive | Free / $3.99/월 | 무제한 | 예쁜 디자인, 카테고리 | 구독 모델, 기능 과다 |
| Atoms | Free | 무제한 | 미니멀, 간결 | 수익화 모델 불명확 |
| **본 프로젝트** | **₩6,500 (1회)** | **3개 (Free)** | **의도적 제약, 극미니멀** | **신규 진입** |

### 차별화 포인트

- **의도적 제약이 곧 기능**: "3개만"이라는 제약 자체가 마케팅 메시지이자 핵심 가치
- **5초 완료 경험**: 앱을 열고 체크하고 나가는 데 5초면 충분
- **1회 결제**: 구독 피로 없이 합리적 가격
- **위젯 우선**: 앱을 열지 않아도 되는 경험 제공
- **크로스 플랫폼**: Expo로 iOS/Android 동시 지원

## 9. 실행 로드맵

### Phase 1: MVP 개발 (2주)

- 메인 화면: 습관 3개 표시 + 원탭 체크
- 습관 등록/수정/삭제
- 스트릭 카운터
- 주간 캘린더 뷰
- AsyncStorage 로컬 저장
- 로컬 알림
- ✅ 성공 기준: 본인이 2주간 매일 사용

### Phase 2: 위젯 + 디자인 Polish (1~2주)

- 홈 위젯 개발 (iOS + Android)
- 체크 시 햅틱 피드백 + 애니메이션
- 다크/라이트 모드
- 온보딩 플로우
- ✅ 성공 기준: 베타 유저 5명 확보 및 피드백

### Phase 3: Pro + 출시 (1~2주)

- RevenueCat IAP 연동 (1회 결제)
- Pro 기능: 무제한 습관, 통계, 테마, 클라우드 백업
- 앱스토어/플레이스토어 제출
- 앱스토어 스크린샷 및 설명 최적화
- ✅ 성공 기준: 스토어 승인 및 첫 다운로드

### Phase 4: 성장 (출시 후)

- Product Hunt 런칭
- 한국/영어 커뮤니티 홍보 (#buildinpublic)
- 사용자 피드백 기반 기능 추가
- 추가 수익: 아이콘 팩, 위젯 테마 등
- ✅ 성공 기준: 100 다운로드, 첫 유료 결제 발생
