## 4. v1.2 — 글로벌 + Android 동시 출시 (작업 중, 2~3주)

> 상세 스프린트 계획: `v1.2-sprint-plan.md`

### 개요
한국어 전용 → **한/영 양 언어 지원** + **Android 정식 출시** 통합 릴리즈.
v1.1(Pro IAP)은 v1.3으로 이연. v1.0.3 → v1.2.0 점프, v1.1 버전 번호는 발행하지 않음.

### 포함 항목

| 영역 | 내용 |
|------|------|
| i18n 라이브러리 | react-i18next + expo-localization, 디바이스 언어 자동 분기 |
| 한/영 카피 | 약 173개 키, 브랜드 용어집 적용 (Flow / Pause / Just 3 / Where small habits grow) |
| 위젯 i18n | iOS Localizable.strings (ko.lproj/en.lproj) + Android values-en/widget_strings.xml |
| 알림 카테고리 i18n | `setupNotificationCategories()`의 "Done" 액션 텍스트 + 알림 본문 |
| 앱 표시명 변경 | `싹 - 습관 트래커` → `싹: 세 가지 습관` / `Ssak: Three Habits` (디바이스 언어 따라 분기) |
| Privacy/Terms 영문판 | 별도 HTML 파일 + `i18n.language` 기반 URL 분기 |
| 메타데이터 영문판 | App Store / Play Store 한/영 모두 등록, ASO 키워드 분석 |
| Android 정식 출시 | Google Play 신규 사업자 계정 필요. 사업자등록 1~3일 + Play Console 검토 1~2주 병렬 |

### 헌법 룰 준수 (변경 없음)
- 룰 1 (3개 제약): 그대로 유지. 영어판도 "Just 3" 제약 동일 적용
- 룰 2 (5초): 다언어 추가가 사용 흐름을 늦추지 않음 — 디바이스 언어 자동 인식, 토글 없음 (룰 6)
- 룰 5 (서버 운영 안 함): i18n 자체는 클라이언트 번들. 변경 없음
- 룰 9 (iOS 우선): 본 시점까지 iOS 전용 운영으로 코어 검증 완료. v1.2에서 Android 동시 출시는 **예외가 아니라 룰 9의 자연스러운 졸업**

### 성공 기준
- 한국어 디바이스: 기존 사용자에게 "이름이 정돈됨" + "잠금화면 액션" 외 변화 없는 매끄러운 업데이트
- 영어 디바이스: 첫 인상에서 어색한 직역 0건, 브랜드 일관성 유지
- Android: iOS와 동등한 사용 경험 (위젯 포함)
- 양 플랫폼 동시 심사 통과
- 본인이 매일 사용 중 (룰 8)

## 5. v1.3 — Pro 도입 (2~3주, 글로벌 출시 후)

### 개요
RevenueCat 실연동으로 1회 결제 Pro 활성화. v1.0 출시 시 일부러 제거했던 기능 복귀 + Pro 가치 추가.
**선행 조건**: v1.2 글로벌 출시 후 양 시장(한국/영어권) 사용 신호 1개월 이상 관찰.

### 포함 기능
| 항목 | 상태 | 구현 포인트 |
|------|------|------------|
| RevenueCat 실연동 | stub 존재 | `src/utils/purchases.ts` stub을 실제 SDK로 교체, App Store / Play Store 상품 등록 |
| Pro 게이트 복귀 | UI 제거됨 | 설정 화면에 Pro 구매/복원 UI 재추가 |
| 통계 화면 | 제거됨 | `app/stats.tsx` 복귀 — 월간/연간 달성률, 최장 흐름 |
| 무제한 습관 | **적용 안 함** | 3개 제약은 헌법 룰 1, Pro에서도 유지. Pro의 가치는 통계/테마/추가 자산으로 |

### 가격
| 티어 | 가격 |
|------|------|
| Free | ₩0 / $0 |
| Pro (Lifetime) | ₩6,500 / $4.99 1회 결제 |

### 성공 기준
- Pro 전환율 5%+
- 한/영 시장 동시 첫 결제 발생
- 결제 플로우 크래시/환불 요청 ≤ 1건
