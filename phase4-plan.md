## 1. v1.0.1 — 출시 후 안정화 (최우선, ~1주)

### 범위
사용자 피드백 수집 + 크래시/UX 미세 조정. 새 기능 추가 금지.

### 태스크
| ID | 항목 | 근거 |
|----|------|------|
| S1 | 위젯 레이아웃 미세조정 커밋 | 현재 `targets/HabitWidget/HabitWidget.swift`에 패딩/간격 수정 미커밋 상태 |
| S2 | 초기 유저 피드백 채널 셋업 | 피드백 이메일은 이미 있음 — 수집 스프레드시트 or 노션 정리 |
| S3 | 크래시/에러 트래킹 확인 | Sentry 미도입 상태라면 간이 도입 검토 (선택) |
| S4 | 스토어 리뷰 모니터링 루틴 | 주 1회 확인, 별 1~3점 우선 대응 |
| S5 | 피드백 기반 버그 핫픽스 | 발견된 것만 처리 |

### 성공 기준
- 1주간 크래시율 1% 이하
- 주 1회 리뷰/피드백 정리
- 기능 추가 없이 유지

## 2. v1.1 — Pro 도입 (2~3주)

### 개요
RevenueCat 실연동으로 1회 결제 Pro 활성화. v1.0 출시 시 일부러 제거했던 기능을 복귀.

### 포함 기능
| 항목 | 상태 | 구현 포인트 |
|------|------|------------|
| RevenueCat 실연동 | stub 존재 | `src/utils/purchases.ts` stub을 실제 SDK로 교체, 상품 등록 |
| Pro 게이트 복귀 | UI 제거됨 | 설정 화면에 Pro 구매/복원 UI 재추가 |
| 통계 화면 | 제거됨 | `app/stats.tsx` 복귀 — 월간/연간 달성률, 최장 스트릭 |
| 무제한 습관 | 로직 존재 | Pro 시 3개 제한 해제 (3개 제약은 Free의 핵심 가치, 룰 1 위배 아님) |

### 가격
| 티어 | 가격 |
|------|------|
| Free | ₩0 |
| Pro | ₩6,500 ($4.99) 1회 결제 |

### 성공 기준
- Pro 전환율 5%+
- 첫 결제 발생
- 결제 플로우 크래시/환불 요청 ≤ 1건

## 3. v1.2 — 스트릭 보상 시스템 (2~3주)

### 개요
연속 달성 시 아이콘/색상 언락 → 꾸미는 재미 학습 → v1.3 유료 팩 전환 유도.

### 보상 테이블
| 연속일수 | 보상 | 무료/유료 | 근거 |
|---------|------|----------|------|
| 3일 | 특별 축하 애니메이션 | 무료 | 초기 이탈 방지 |
| 7일 | 아이콘 1개 | 무료 | 첫 주간 사이클 완성 |
| 14일 | 색상 1개 | 무료 | 의식적 노력기 종료 |
| 21일 | 아이콘 2개 + 색상 1개 | 무료 | 21일 법칙 |
| 30일 | 테마 미리보기 | 열람 무료 / 적용 유료 | 전환점, 구매 욕구 자극 |
| 50일 | 위젯 스킨 미리보기 | 열람 무료 / 적용 유료 | 결제 의향 최고 |
| 100일 | 한정 아이콘 "백일장" | 무료 | 핵심 유저 충성도 |
| 365일 | 한정 아이콘 "1주년" | 무료 | 감사 표현 |

### 스트릭 계산 방식
- **이어가기 시스템과 충돌 주의**: 현재 "쉼표" 로직은 하루 빠져도 흐름 유지. 보상 기준은 "이어가기 포함 스트릭" vs "순수 연속 스트릭" 중 선택 필요 — v1.1 출시 후 논의.

### UX 원칙 (헌법 준수)
- 메인 화면에 보상 관련 UI 추가 금지 (룰 3 — 빼는 것 먼저)
- 새 화면 만들지 않음 — 기존 아이콘/색상 피커에 잠금 상태만 추가
- 축하 오버레이 확장으로만 보상 알림
