## 5. v1.3 — 졸업 시스템 + 통계 화면 (1.5~2주)

### 개요
"습관이 자리잡았다"는 자기 인식을 시스템에 반영. 50일(개화) 도달 시 자동 제안 + 7일 이후 본인 의지로 조기 졸업 가능 + 졸업한 정원 = **통계 화면 안에 호스트** (통계 화면 복귀와 묶음).
**원래 v1.3 = Pro IAP + 통계 화면이었으나 2026-05-13 재정렬**: 사용자가 v1.2 출시 전에 조기 졸업 필요성을 직접 도출 → 룰 4 한시 예외 적용 + 통계 화면 동반 복귀 (졸업한 정원 호스트). Pro IAP는 v1.4로 이연.

상세 sprint 계획은 `v1.3-sprint-plan.md` 참조.

### 디자인 안 (확정)

**편집 화면 액션 노출 (단계별 분기)**:
| 진행 일수 | 노출되는 액션 |
|----------|--------------|
| 0~6일 (씨앗/발아) | `[삭제]` 만 |
| 7~49일 (떡잎/줄기/꽃봉오리) | `[삭제]` + `[조기 졸업]` |
| 50일 이상 (개화) | `[삭제]` + `[졸업시키기]` |

**50일 자동 제안 (1회만)**:
- 개화 도달 후 다음 체크 시점에 다이얼로그 발동
- "🌸 꽃이 피었어요. 새 씨앗을 심을까요?" / `[보내주기]` `[계속 키울게요]`
- 거절 후 다시 자동 제안 없음 — 본인 의지로 편집 화면에서 언제든 졸업 가능

**조기 졸업 확인 다이얼로그**:
- "조기 졸업할까요? 개화까지 N일 남았어요" / `[네, 졸업시킬게요]` `[취소]`
- 50일 미만에서만 한 번 더 확인

**통계 화면 (Free 노출)**:
- `app/stats.tsx` 신규 또는 복귀
- 기본 통계: 활성 습관 / 총 완료 / 최장 흐름 / 해금 수
- **졸업한 정원 섹션**: 졸업 항목 카드들 (이름 / 아이콘 / 졸업 일자 / 총 흐름 / 단계)
- settings 화면에서 진입 (메인 화면 변경 없음, 룰 3 준수)

**액션 차별화**:
| 액션 | 의미 | 시각 |
|------|------|------|
| 삭제 | "더 이상 추적 안 함" | 빨강, 경고 톤 |
| 졸업 | "내 안에 자리잡았어요" | 초록 + 🌸, 보상 톤 |

### 데이터 모델 확장

`Habit` 인터페이스에 추가:
- `graduatedAt?: string` — YYYY-MM-DD
- `graduatedStage?: GrowthStageId` — 졸업 시 단계 (떡잎/줄기/꽃봉오리/개화)
- `totalFlowDays?: number` — 졸업 시점 누적 흐름 일수
- `isGraduated?: boolean` — 졸업 상태 플래그 (활성 슬롯 카운트에서 제외)

활성 슬롯 카운트는 `isGraduated !== true` 기준 — 룰 1 (3개 제약) 활성에만 적용.

### 헌법 룰 적합성

- **룰 1 (3개 제약)**: 활성 슬롯 3개 유지. 졸업한 정원은 별도 컬렉션 = 활성 외 → 본질 훼손 없음
- **룰 2 (5초)**: 자동 제안 + 확인 다이얼로그 모두 단일 탭 결정
- **룰 3 (빼기 먼저)**: 신규 화면 = 통계 화면 1개. 졸업한 정원은 통계 안에 호스트 — 별도 화면 만들지 않음
- **룰 4 (로드맵 외 금지)**: 본인이 사용 경험에서 직접 도출 → 한시 예외 (2026-05-13)
- **룰 6 (설정 안 늘림)**: 임계값(50일/7일) 시스템 고정, 사용자 설정 불가. 동적 라벨로 두 버튼 통합
- **룰 8 (본인 매일 사용)**: 본인 판단 신뢰. 시스템이 막지 않음 (조기 졸업 7일 이후 자유)
- **룰 10 (존재 이유)**: 답 = "달성 → 새 도전 사이클을 자연스럽게 만들어 3개 제약을 지속 가능하게 함"

### 폐기된 옵션

- ❌ 24시간 갈아치우기 방지 룰 — 3개 동시 50일 도달이 디폴트 경로라 자연스러운 시나리오 방해. 룰 8 위배
- ❌ 사용자 설정 임계값 — 룰 6 위배. "각자 다른 기준" 문제는 거절 권리로 해결
- ❌ 졸업한 정원을 별도 풀스크린 화면 — 룰 3 위배 (통계 화면 안에 흡수)
- ❌ 통계 화면을 Pro 전용 게이트 — 졸업한 정원 호스트라 Free 노출 필수

| 50일 이상 (개화) | `[삭제]` + `[졸업시키기]` |

- 동적 라벨(`조기 졸업` ↔ `졸업시키기`)로 정시 vs 조기 인식 차이 표현 — 룰 6(설정 안 늘림) 준수

**50일 자동 제안 (1회만)**:

- "🌸 꽃이 피었어요. 새 씨앗을 심을까요?" / `[보내주기]` `[계속 키울게요]`
- `[계속]` 선택 시 다시 자동 제안 없음 → 본인 의지로 편집 화면에서 언제든 졸업 가능 (발견성 보장)

**조기 졸업 확인 다이얼로그**:
- "조기 졸업할까요? 개화까지 N일 남았어요" / `[네, 졸업시킬게요]` `[취소]`
- 50일 미만에서만 한 번 더 확인 — 즉흥적 조기 졸업 방지

**졸업한 정원 (v1.3 통계 화면 확장)**:
- 졸업 항목 카드: 이름 / 아이콘 / 졸업 일자 / 총 흐름 일수 / 졸업 단계 (떡잎/줄기/꽃봉오리/개화)
- "내가 어디서 멈췄는지" 가시화 — 단순 삭제와 차별화
- v1.3 통계 화면 복귀 시 자리만 미리 잡아두고 v1.5에서 콘텐츠 채움

**액션 차별화 (UI)**:
| 액션 | 의미 | 시각 |

### 헌법 룰 적합성

- **룰 1 (3개 제약)**: 활성 슬롯 3개 유지. 졸업한 정원은 별도 컬렉션 = 활성 외 → 본질 훼손 없음
- **룰 2 (5초)**: 자동 제안 + 확인 다이얼로그 모두 단일 탭으로 결정. 5초 안에 종료

- **룰 10 (존재 이유)**: 답 = "달성 → 새 도전 사이클을 자연스럽게 만들어 3개 제약을 지속 가능하게 함"
