## 3. v1.2 — 싹 성장 시스템 + 스트릭 보상 (3주)

### 개요
**싹 성장 시각화를 기반 레이어로**, 그 위에 기존 아이콘/색상 해금 시스템을 얹는 통합 설계.
체크할 때마다 싹이 자라고, 특정 성장 단계 도달 시 해금 이벤트가 발생.
피드백 FB-002(브랜드 통합 축하), FB-003(해금 사전 안내), FB-004(진행도 힌트) 세 건을 하나의 시스템으로 수렴.

### 설계 원칙 (2026-04-19 회의 확정)
- **숫자 없음, 싹이 먼저**: 진행도는 싹 성장 단계로 표현. "N일 남음" 같은 숫자는 임박 시에만 예외적 속삭임
- **외재적 동기 최소화**: 보상이 목적이 아니라 습관이 목적. 보상은 결과적 수반물
- **기존 해금 자산 재활용**: `src/constants/rewards.ts`의 7/14/30/50일 티어를 성장 단계와 매핑 (재설계 금지)
- **이어가기 시스템 존중**: 퇴행 로직은 "이틀 연속 미완료 시에만" 적용 — "하루 쉬어도 괜찮아요" 톤 유지

### 싹 성장 단계 ↔ 해금 티어 매핑 (제안)

| 성장 단계 | 달성 조건 | 해금 (기존 `rewards.ts` 티어) | 시각 |
|----------|----------|------------------------------|------|
| 씨앗 | 0~2일 | - | 🌰 (정지) |
| 발아 | 3일 | - (마이크로 축하만) | 🌱 |
| 떡잎 | 7일 | 색상 2개 (기존 티어 1) | 🌿 |
| 줄기 | 14일 | 아이콘 4개 (기존 티어 2) | 🎋 |
| 꽃봉오리 | 30일 | 프리미엄 색상 3개 (기존 티어 3) | 🌷 |
| 개화 | 50일 | 프리미엄 아이콘 4개 (기존 티어 4) | 🌸 |

### 임박 힌트 규칙 (숫자 최소화)
- **4일 이상 남음**: 힌트 없음. 싹 자체가 진행도
- **3일 이하 남음**: 체크 완료 오버레이에 작게 "다음 성장까지 N일 🌿" 1회
- **해금 당일**: 기존 `UnlockToast` 리디자인된 버전 발동 (성장 단계 시각 포함)

### 구현 체크리스트
- [x] `CelebrationOverlay` 리팩터: 컨페티 제거 → 싹 성장 애니메이션 (1~1.5초) — 2026-04-24
- [x] `UnlockToast` 리디자인: 🎉 → 성장 단계 시각, 크림/세이지 톤 재적용 — 2026-04-24
- [x] 성장 단계 상태 관리: 파생값으로 처리 — `src/constants/growth.ts` 헬퍼만 추가, 별도 store 불필요
- [x] 습관별 성장 상태 vs 전체 최장 흐름 기반: **전체 최장 흐름** 채택 (기존 `rewardStore.maxFlowEver`와 일관성)
- [x] 에셋: 이모지 기반 (🌰🌱🌿🎋🌷🌸) — SVG 보류 (MVP 우선)
- [ ] 퇴행 로직: 이틀 연속 미완료 시 1단계 퇴행 — **v1.2.1로 분리** (이어가기 시스템과 충돌 검토 필요)

### 보류 항목 (v1.x 전체에서 보류)
- 100일·365일 한정 아이콘 — 기존 테이블에 존재했으나, MVP 단계에선 50일 이후 엔드게임 미구현
- 21일 중간 단계 — 7·14·30일 흐름이 이미 충분히 촘촘하다고 판단, 복잡도 축소

### 스트릭 계산 방식
- **이어가기 시스템과 충돌 주의**: 현재 "쉼표" 로직은 하루 빠져도 흐름 유지. 보상 기준은 "이어가기 포함 스트릭" vs "순수 연속 스트릭" 중 선택 필요 — v1.1 출시 후 논의.

### UX 원칙 (헌법 준수)
- 메인 화면에 보상 관련 UI 추가 금지 (룰 3 — 빼는 것 먼저)
- 새 화면 만들지 않음 — 기존 아이콘/색상 피커에 잠금 상태만 추가
- 축하 오버레이 확장으로만 보상 알림
