## OLD-5. (구) v1.3 — 졸업 시스템 + 통계 화면 (v1.1.0에 흡수)

### 개요
"습관이 자리잡았다"는 자기 인식을 시스템에 반영. 50일(개화) 도달 시 자동 제안 + 7일 이후 본인 의지로 조기 졸업 가능 + 졸업한 정원 = **통계 화면 안에 호스트** (통계 화면 복귀와 묶음).
**원래 v1.3 = Pro IAP + 통계 화면이었으나 2026-05-13 재정렬**: 사용자가 v1.2 출시 전에 조기 졸업 필요성을 직접 도출 → 룰 4 한시 예외 적용 + 통계 화면 동반 복귀 (졸업한 정원 호스트). Pro IAP는 v1.4로 이연.

상세 sprint 계획은 `v1.3-sprint-plan.md` 참조.

### 디자인 안 (확정)

**편집 화면 액션 노출 (단계별 분기)**:
| 진행 일수 | 노출되는 액션 |
|----------|--------------|
| 0~6일 (씨앗/발아) | `[삭제]` 만 |
| 7~49일 (떡잎/줄기/꽃봉오리) | `[삭제]` + `[조기 졸업]` |
| 50일 이상 (개화) | `[삭제]` + `[졸업시키기]` |

**50일 자동 제안 (1회만)**:
- 개화 도달 후 다음 체크 시점에 다이얼로그 발동
- "🌸 꽃이 피었어요. 새 씨앗을 심을까요?" / `[보내주기]` `[계속 키울게요]`
- 거절 후 다시 자동 제안 없음 — 본인 의지로 편집 화면에서 언제든 졸업 가능

**조기 졸업 확인 다이얼로그**:
- "조기 졸업할까요? 개화까지 N일 남았어요" / `[네, 졸업시킬게요]` `[취소]`
- 50일 미만에서만 한 번 더 확인

### 데이터 모델 확장

`Habit` 인터페이스에 추가:
- `graduatedAt?: string` — YYYY-MM-DD
- `graduatedStage?: GrowthStageId` — 졸업 시 단계 (떡잎/줄기/꽃봉오리/개화)
- `totalFlowDays?: number` — 졸업 시점 누적 흐름 일수
- `isGraduated?: boolean` — 졸업 상태 플래그 (활성 슬롯 카운트에서 제외)

활성 슬롯 카운트는 `isGraduated !== true` 기준 — 룰 1 (3개 제약) 활성에만 적용.

### 헌법 룰 적합성

- **룰 1 (3개 제약)**: 활성 슬롯 3개 유지. 졸업한 정원은 별도 컬렉션 = 활성 외 → 본질 훼손 없음
- **룰 2 (5초)**: 자동 제안 + 확인 다이얼로그 모두 단일 탭 결정
- **룰 3 (빼기 먼저)**: 신규 화면 = 통계 화면 1개. 졸업한 정원은 통계 안에 호스트 — 별도 화면 만들지 않음

## 5. v1.4 — 수익화 (신규 모델, 2026-06-25 전면 재정의)

> ⚠️ 종전 "프리미엄(무료+Pro ₩6,500, 통계 주력)" 안은 **폐기**. 아래가 최신.
> 동기 = 꾸미기만으론 *의미가 약해* 비용 충당(전환)조차 실패한다는 판단 → 의미(백업·손실회피)가 있어야 지갑이 열림.

### 개요
**무료 체험 → 1회 결제(유료 전환)** 모델. **백업이 주력 가치, 꾸미기는 덤.**
trial + 백업 + 결제를 한 버전(v1.4)에 묶음 — 백업이 "의미"라 쪼개면 약해지고, 백업을 먼저 무료로 풀면 과금 시 *클로백(준 걸 뺏기)* 이 되므로 분리 불가.

### 모델
| 항목 | 내용 |
|------|------|
| 체험 | **14일 기간 게이트.** 만료 시 계속 쓰려면 결제 |
| 가격 | 1회 결제 **₩3,000~4,400 검토** (백업 포함이라 종전 단순 꾸미기보다 가격 정당화 폭 큼) |
| 권유 타이밍 | 감정 고점 — **첫 7일 흐름(첫 떡잎) 직후 + 만료 직전** (게이트는 시간, 파는 건 streak 고점) |
| 포함 | **백업/복원(주력)** + 모든 꾸미기(테마·앱아이콘·응원자 색) |
| 백업 구현 | OS 기본 — iOS iCloud/CloudKit, Android Auto Backup. 직접 서버 X(룰 5), 인프라비 0 → 1회 결제와 적자 미스매치 없음 |
| 무제한 습관 | **적용 안 함** (룰 1 — 유료로도 3개 유지) |
| 아이콘/색상 | 아이콘 전면 무료(이모지 자유 선택) + 색상=흐름 보상. **2026-06-25 이미 구현·커밋** (`403bf12`) |

### 선행 작업
- **prebuild** (IAP·네이티브 백업 공통 선행)
- RevenueCat 실연동 (`src/utils/purchases.ts` stub 교체) + 스토어 상품 등록 + 구매/복원 UI
- 14일 트라이얼 상태·만료 로직 + 권유 트리거(첫 7일 흐름 / 만료 직전)
- iOS CloudKit/iCloud 컨테이너 + Android Auto Backup + 복원 플로우 + 데이터 스키마/마이그레이션
- **헌법 1번 문구 결정 (오너)**: 1번이 "Free 버전의 3개" 명시 → trial→paid는 그 *문구*와 충돌. 신성한 핵심(**3개 제한**)은 유지, "영구 무료 티어 존재" 전제만 바뀜. 전문상 출시 전 개정 금지라 의식적 결정 필요.

### 통계 확장은 v1.5로 이연
종전 v1.4 Pro 가치였던 "월/연 통계·그래프·CSV"는 더 이상 Pro 핵심이 아니므로 v1.5 이후 enhancement로 이연.

### 성공 기준
- 비용 충당(연 ~$99 ≈ 1회 ₩3,000 × 50~65건) — 의미(백업) 있어야 달성
- 한/영 첫 결제 발생, 결제/복원 크래시·환불 ≤ 1건
- 본인이 기기 변경/복원을 1회 이상 실제로 써봄 (룰 8)
