## 2주 일별 스프린트 배치

### Week 1: 핵심 플로우 완성 (습관 체크가 동작하는 상태)

| 일차 | 태스크 | 목표 |
|------|--------|------|

| **Day 3 (수)** | T4 + T7 | **핵심 마일스톤: 습관 체크가 실제로 동작하고 저장됨** |

## 각 태스크 완료 조건

### T1: Expo 프로젝트 초기 세팅
- [ ] `npx create-expo-app` 실행, TypeScript 템플릿 적용
- [ ] Expo Router 파일 기반 라우팅 설정 (app/ 디렉토리)
- [ ] Zustand + Nativewind v4 설치 및 설정
- [ ] 폴더 구조 확정: `app/` (라우팅) + `src/components`, `src/store`, `src/utils`, `src/types`, `src/constants`
- [ ] 앱 실행 시 빈 메인 화면 표시 확인 (iOS 시뮬레이터)

### T2: 데이터 모델 + AsyncStorage CRUD
- [ ] `Habit`, `HabitLog` TypeScript 타입 정의 (기획서 데이터 구조 준수)
- [ ] `habitStorage.ts`: 습관 CRUD 함수 4개 (create, read, update, delete)
- [ ] `habitLogStorage.ts`: 로그 기록/조회 함수 (오늘 체크, 기간별 조회)
- [ ] 각 함수 단독 호출 시 정상 동작 확인

### T3: 메인 화면 레이아웃
- [ ] 습관 카드 3개가 세로로 표시됨 (하드코딩 데이터)
- [ ] 각 카드에 습관 이름, 아이콘, 색상 표시
- [ ] 체크 버튼 영역 존재 (아직 동작 안 해도 됨)

### T4: 원탭 체크 기능
- [ ] 습관 카드 탭 시 완료/미완료 토글
- [ ] 완료 상태가 시각적으로 구분됨 (색상 변화 등)
- [ ] 완료 상태가 AsyncStorage에 저장됨
- [ ] 앱 재시작 후에도 오늘의 체크 상태 유지

### T5: 습관 등록 화면
- [ ] 습관 이름 텍스트 입력
- [ ] 아이콘 선택 (프리셋 8~12개)
- [ ] 색상 선택 (프리셋 6~8개)
- [ ] 저장 버튼 탭 시 AsyncStorage에 저장 후 메인 화면 복귀
- [ ] 빈 이름으로는 저장 불가

### T6: 습관 수정/삭제
- [ ] 메인 화면에서 습관 롱프레스 또는 편집 진입점 존재
- [ ] 수정 시 기존 데이터 로드, 변경 후 저장
- [ ] 삭제 시 확인 다이얼로그 표시 후 삭제
- [ ] 삭제 시 관련 HabitLog도 함께 삭제

### T7: 메인 화면 실제 데이터 연결
- [ ] 앱 시작 시 AsyncStorage에서 습관 목록 로드
- [ ] 습관 없을 때 빈 상태 안내 표시
- [ ] 체크 상태가 실시간 반영

### T8: 3개 습관 제한
- [ ] 습관이 3개일 때 등록 버튼 비활성화 또는 안내 메시지
- [ ] 어떤 경로로도 4개째 습관 등록 불가 (헌법 룰 1)

### T9: 스트릭 카운터 로직
- [ ] 오늘 기준 역순으로 연속 완료 일수 계산
- [ ] 하루라도 빠지면 스트릭 리셋 (0부터)
- [ ] 오늘 미완료 시 어제까지의 스트릭 표시
- [ ] 엣지 케이스: 습관 생성 당일, 연속 0일, 연속 100일+

### T10: 스트릭 카운터 UI
- [ ] 메인 화면 각 습관 카드에 스트릭 숫자 표시
- [ ] "N일 연속" 형식

### T11: 주간 캘린더 뷰 UI
- [ ] 이번 주 월~일 7일 그리드 표시
- [ ] 오늘 날짜 하이라이트
- [ ] 각 날짜 셀에 달성 상태 표시 영역

### T12: 주간 캘린더 데이터 연결
- [ ] 각 날짜별 습관 달성 여부 시각적 표시 (완료/미완료/해당없음)
- [ ] 습관별 색상으로 구분
- [ ] 이번 주 달성률 한눈에 파악 가능

### T13: 로컬 알림 설정
- [ ] expo-notifications 설치 및 권한 요청
- [ ] 습관별 지정 시간에 로컬 푸시 알림 발송
- [ ] 앱 종료 상태에서도 알림 수신
- [ ] iOS 시뮬레이터/실기기에서 동작 확인

### T14: 알림 시간 설정 UI
- [ ] 습관 등록/수정 화면에 알림 시간 선택 추가
- [ ] 시간 선택 시 알림 스케줄 등록
- [ ] 알림 끄기 옵션
- [ ] 기존 습관의 알림 시간 변경 시 이전 스케줄 취소 후 재등록

### T15: 통합 테스트 + 버그 수정
- [ ] 전체 플로우 수동 테스트: 등록 -> 체크 -> 스트릭 확인 -> 캘린더 확인
- [ ] 앱 재시작 후 데이터 유지 확인
- [ ] 3개 제한 우회 불가 확인
- [ ] 알림 정상 수신 확인
- [ ] 크래시/에러 없이 주요 플로우 완주
